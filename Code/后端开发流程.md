## 一、开发一个新功能（完整 6 步）

以「新增评论功能」为例，演示完整流程。

---

### 第 1 步：创建 DTO

**位置：** `Dtos/<模块名>/`

定义前端传什么、后端返回什么。

```
Dtos/
└── Review/
    └── ReviewDtos.cs
```

```csharp
namespace Backend.Dtos.Review;

// 前端创建评论时传的 JSON
public class CreateReviewDto
{
    public int Rating { get; set; }         // 评分 1-5
    public string? Info { get; set; }       // 评价内容
    public long PurchaseId { get; set; }    // 订单ID
}

// 后端返回给前端的 JSON
public class ReviewDto
{
    public int ReviewId { get; set; }
    public int Rating { get; set; }
    public string? Info { get; set; }
    public DateTime ReviewTime { get; set; }
    public long PurchaseId { get; set; }
    public List<ReviewImageDto> Images { get; set; } = new();
}

public class ReviewImageDto
{
    public long ImgId { get; set; }
    public string ImgUrl { get; set; } = string.Empty;
    public int ImgIndex { get; set; }
}
```

**原则：**
- `XXXDto` — 后端返回给前端（可隐藏敏感字段）
- `CreateXXXDto` — 前端创建时传（只含必要字段）
- `UpdateXXXDto` — 前端修改时传（字段都可选）

### 第 2 步：创建 Repository

**位置：** `Repositories/`

每个 Repository = 一个接口 + 一个实现。

```
Repositories/
├── IReviewRepository.cs      ← 接口
└── ReviewRepository.cs       ← 实现
```

**接口（定义能做什么）：**

```csharp
public interface IReviewRepository
{
    Task<Review?> GetByIdAsync(int reviewId);
    Task<List<Review>> GetAllAsync();
    Task<Review?> GetByPurchaseIdAsync(long purchaseId);
    Task AddAsync(Review review);
    void Update(Review review);
    void Delete(Review review);
    Task SaveAsync();
}
```

**实现（具体怎么做）：**

```csharp
public class ReviewRepository : IReviewRepository
{
    private readonly AppDbContext _context;
    public ReviewRepository(AppDbContext context) => _context = context;

    public async Task<Review?> GetByIdAsync(int reviewId)
        => await _context.Reviews
            .Include(r => r.Images)
            .FirstOrDefaultAsync(r => r.ReviewId == reviewId);

    public async Task<Review?> GetByPurchaseIdAsync(long purchaseId)
        => await _context.Reviews
            .FirstOrDefaultAsync(r => r.PurchaseId == purchaseId);

    public async Task AddAsync(Review review)
        => await _context.Reviews.AddAsync(review);

    public void Update(Review review)
        => _context.Reviews.Update(review);

    public void Delete(Review review)
        => _context.Reviews.Remove(review);

    public async Task SaveAsync()
        => await _context.SaveChangesAsync();
    // 其余方法类似...
}
```

**原则：** Repository 只做数据存取，不写业务逻辑。

### 第 3 步：创建 Service

**位置：** `Services/`

Service 写业务规则，调用 Repository。

```
Services/
├── IReviewService.cs         ← 接口
└── ReviewService.cs          ← 实现
```

```csharp
public class ReviewService : IReviewService
{
    private readonly IReviewRepository _reviewRepo;
    private readonly IPurchaseRepository _purchaseRepo;

    public ReviewService(IReviewRepository reviewRepo, IPurchaseRepository purchaseRepo)
    {
        _reviewRepo = reviewRepo;
        _purchaseRepo = purchaseRepo;
    }

    public async Task<ReviewDto> CreateAsync(CreateReviewDto dto)
    {
        // 业务规则1：检查订单是否存在
        var purchase = await _purchaseRepo.GetByIdAsync(dto.PurchaseId);
        if (purchase == null)
            throw new Exception("订单不存在");

        // 业务规则2：一个订单只能评价一次
        var exist = await _reviewRepo.GetByPurchaseIdAsync(dto.PurchaseId);
        if (exist != null)
            throw new Exception("该订单已评价");

        // 业务规则3：只有完成的订单才能评价
        if (purchase.Status != "success")
            throw new Exception("订单未完成，无法评价");

        var review = new Review
        {
            Rating = dto.Rating,
            Info = dto.Info,
            PurchaseId = dto.PurchaseId,
            ReviewTime = DateTime.Now
        };

        await _reviewRepo.AddAsync(review);
        await _reviewRepo.SaveAsync();

        return ToDto(review);
    }

    private static ReviewDto ToDto(Review r) => new()
    {
        ReviewId = r.ReviewId,
        Rating = r.Rating,
        Info = r.Info,
        ReviewTime = r.ReviewTime,
        PurchaseId = r.PurchaseId
    };
}
```

**原则：** Service = 业务判断的集中地，所有 `if/else/throw` 都在这里。

### 第 4 步：创建 Controller

**位置：** `Controllers/`

Controller = 对外暴露的 HTTP 接口。

```csharp
[ApiController]
[Route("api/[controller]")]                    // → /api/review
public class ReviewController : ControllerBase
{
    private readonly IReviewService _reviewService;

    public ReviewController(IReviewService reviewService)
    {
        _reviewService = reviewService;
    }

    [HttpGet]                                   // GET /api/review
    public async Task<ActionResult<List<ReviewDto>>> GetAll()
    {
        var reviews = await _reviewService.GetAllAsync();
        return Ok(reviews);
    }

    [HttpGet("{id}")]                           // GET /api/review/1
    public async Task<ActionResult<ReviewDto>> GetById(int id)
    {
        var review = await _reviewService.GetByIdAsync(id);
        if (review == null) return NotFound();
        return Ok(review);
    }

    [HttpPost]                                  // POST /api/review
    public async Task<ActionResult<ReviewDto>> Create([FromBody] CreateReviewDto dto)
    {
        try
        {
            var review = await _reviewService.CreateAsync(dto);
            return CreatedAtAction(nameof(GetById), new { id = review.ReviewId }, review);
        }
        catch (Exception ex)
        {
            return BadRequest(new { error = ex.Message });
        }
    }
}
```

**原则：** Controller 只收发 HTTP，一行业务代码都不写。

### 第 5 步：注册 DI（依赖注入）

**位置：** `Program.cs`

在对应区域加两行：

```csharp
// ==================== Repository 层注册 ====================
builder.Services.AddScoped<IReviewRepository, ReviewRepository>();   // ← 加这行

// ==================== Service 层注册 ====================
builder.Services.AddScoped<IReviewService, ReviewService>();          // ← 加这行
```

**原则：** 每个新 Repository 和新 Service 都要注册，漏了编译报错会提醒你。

## 二、开发规范速查

### 文件命名规则

| 层 | 接口 | 实现 |
|----|------|------|
| Controller | — | `XxxController.cs` |
| Service | `IXxxService.cs` | `XxxService.cs` |
| Repository | `IXxxRepository.cs` | `XxxRepository.cs` |

### 每层只能做什么

| 层 | 可以 | 不可以 |
|----|------|--------|
| **Controller** | 接收请求，调 Service，返回 HTTP 状态码 | 写 if/else 业务判断，直接调 Repository |
| **Service** | 写业务逻辑，调 Repository，抛异常 | 直接访问 HttpContext，直接操作 DbContext |
| **Repository** | 调 DbContext，执行 CRUD | 写业务规则，调其他 Repository |
| **DbContext** | 连数据库 | 写逻辑 |

### 依赖注入规则

```
Controller  依赖 → Service 接口（不依赖实现类）
Service     依赖 → Repository 接口（不依赖实现类）
Repository  依赖 → DbContext
```

**绝不允许**：Controller 直接调 Repository，或 Service 直接调 DbContext。

### 返回前端的数据

- ✅ 返回 `DTO`（可控制哪些字段暴露）
- ❌ 直接返回 `Model`（会暴露密码 hash、is_banned 等敏感字段）
- ❌ 直接返回匿名对象 `new { ... }`

### 异常处理

```csharp
// ✅ Service 里抛异常，Controller 统一 catch
public async Task<ReviewDto> CreateAsync(CreateReviewDto dto)
{
    if (某某条件不满足)
        throw new Exception("明确的中文错误信息");
    // 正常逻辑...
}

// ✅ Controller 里统一捕获
try
{
    var result = await _service.CreateAsync(dto);
    return Ok(result);
}
catch (Exception ex)
{
    return BadRequest(new { error = ex.Message });
}
```

## 四、手写 SQL（按需使用）

```csharp
// 复杂查询时可以用原生 SQL
public async Task<List<Product>> SearchAsync(string keyword)
{
    return await _context.Products
        .FromSqlRaw(@"SELECT * FROM ""product"" 
                      WHERE ""name"" LIKE '%' || :kw || '%' 
                      OR ""info"" LIKE '%' || :kw || '%'",
            new OracleParameter(":kw", keyword))
        .ToListAsync();
}
```

## 六、常见问题

**Q: 启动报错 "找不到表"？**
A: 先跑 `dotnet ef database update` 同步数据库。

**Q: 忘了注册 DI，怎么发现？**
A: 编译会报错或启动时报 `Unable to resolve service`。

**Q: 怎么知道数据库里有没有这张表？**
A: SSH 进服务器，`docker exec -it oracle23 sqlplus ...`，然后 `SELECT * FROM user_tables;`
